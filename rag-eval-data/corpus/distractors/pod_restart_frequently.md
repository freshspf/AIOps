# 干扰分析：Pod 频繁重启问题

在 Kubernetes 环境中，Pod 频繁重启会导致服务不可用，影响用户体验。频繁重启不仅会消耗资源，还可能导致数据丢失和运维告警疲劳。

## 现象特征
1. **Pod 健康检查失败**：Readiness 或 Liveness 探针失败，导致 Pod 被反复重启。
2. **重启次数激增**：通过 `kubectl get pods` 或监控面板看到 Pod 重启计数快速增长。
3. **服务不稳定**：部分请求成功，部分请求失败，表现为间歇性 502/503/504。

## 常见原因
1. **OOM Kill**：容器内存超限被内核强制杀死。
2. **探针失败**：应用启动慢导致 Liveness 失败，或业务线程死锁导致 Readiness 失败。
3. ** CrashLoopBackOff**：应用进程异常退出（如未捕获异常、OOM Panic）。
4. **资源争抢**：多个 Pod 争抢节点资源，导致资源不足被驱逐。
5. **健康检查配置不当**：探针阈值设置过严或超时时间过短。

## 排查建议
1. 查看 Pod 事件：`kubectl describe pod <pod-name>`，重点关注 Last State 和 Events。
2. 查看应用日志：检查是否有 OOM、Panic 或未捕获异常。
3. 检查资源限制：确认 CPU/Limits 和 Memory Limits 配置是否合理。
4. 调整探针参数：适当增加启动时间、超时时间和失败重试次数。
