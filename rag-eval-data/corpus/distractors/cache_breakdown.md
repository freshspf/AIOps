# 干扰分析：缓存击穿与雪崩

缓存在高并发系统中承担着重要的性能角色，但使用不当会导致"缓存击穿"和"缓存雪崩"等问题，瞬间压垮数据库或下游服务。

## 核心概念

### 缓存击穿（Cache Breakdown）
指某个热点 key 过期瞬间，大量并发请求同时穿透到数据库，导致数据库压力剧增。
- **现象**：单个 key 失效后，数据库 QPS 瞬间飙升。
- **原因**：热点 key 过期或不存在，且无保护机制。

### 缓存雪崩（Cache Avalanche）
指大量缓存 key 同时过期，或者缓存服务整体不可用，导致大量请求同时打到数据库。
- **现象**：大量接口同时变慢或超时，数据库连接池耗尽。
- **原因**：缓存服务重启、批量 key 过期、缓存集群故障。

## 现象特征
1. **数据库压力骤增**：数据库 CPU 和连接数突然暴涨。
2. **响应时间激增**：接口 P99 响应时间从正常值飙升到数秒。
3. **影响范围广**：大量接口同时变慢，而非单一接口。

## 排查建议
1. 检查缓存命中率：确认是否存在大量缓存未命中。
2. 分析过期策略：是否存在大量 key 集中过期的情况。
3. 检查缓存服务状态：Redis/Memcached 是否正常，是否有集群故障。
4. 验证热点 key：是否存在极端热点 key 且无保护。

## 防护策略
- 使用互斥锁或分布式锁保护热点 key。
- 对缓存过期时间加随机值，避免批量过期。
- 实现缓存预热，预加载热点数据。
- 设计多级缓存架构，降低对单点缓存的依赖。
