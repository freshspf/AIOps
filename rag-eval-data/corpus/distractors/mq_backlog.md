# 干扰分析：消息队列积压问题

消息队列（MQ）是分布式系统中实现异步通信和解耦的核心组件。当消费能力跟不上生产速度时，消息会在队列中不断积压，最终导致系统性能下降或服务不可用。

## 现象特征
1. **队列深度激增**：监控面板显示消息队列的积压数量快速上涨。
2. **消费延迟增大**：消息从生产到消费的耗时（Latency）明显变长。
3. **消费端超时**：消费者处理单条消息耗时过长，导致频繁超时或重试。

## 常见原因
1. **消费能力不足**：消费者实例数量少或单消费者处理能力有限。
2. **消费逻辑阻塞**：消费者代码中存在慢查询、远程调用卡死或死循环。
3. **生产端突发流量**：营销活动或突发流量导致消息短时间内大量涌入。
4. **分区/队列不均**：某些分区或队列消息过多，其他分区空闲（分区不均）。

## 排查建议
1. 检查消费者实例的 CPU、内存、网络是否正常。
2. 查看消费者日志，定位耗时最长的处理步骤。
3. 检查生产端是否有异常的批量消息发送。
4. 分析消息大小，是否存在巨型消息导致单条处理耗时过长。
