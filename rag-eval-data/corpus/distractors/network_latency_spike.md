# 干扰分析：网络延迟抖动问题

网络延迟抖动（Network Latency Spike）是分布式系统中常见但难以排查的问题。它可能导致服务间的 RPC 调用超时、数据库连接异常、甚至触发服务雪崩。

## 现象特征
1. **间歇性超时**：服务间调用偶尔出现超时，但大部分时间正常。
2. **跨服务传播**：网络抖动会影响所有依赖该网络路径的服务，导致多个服务同时出现响应变慢。
3. **无明显异常**：应用服务器和数据库本身的 CPU、内存、磁盘都正常，但响应就是慢。

## 常见原因
1. **网络设备故障**：交换机、路由器、负载均衡器等网络设备异常或丢包。
2. **带宽占满**：某个大文件传输或备份任务占满网卡带宽。
3. **DNS 解析问题**：DNS 解析超时或解析到异常 IP。
4. **跨可用区/跨地域调用**：跨机房或跨地域的网络路径更长，抖动更明显。
5. **TCP 重传**：网络丢包导致 TCP 多次重传，延迟增加。

## 排查建议
1. 使用 `ping` 和 `traceroute` 检查网络路径和延迟。
2. 检查网络设备的监控指标（带宽、丢包率、错误包）。
3. 分析是否有大流量任务在非高峰期运行。
4. 检查 DNS 解析是否正常，必要时使用 IP 直连。
