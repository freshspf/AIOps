# 干扰分析：Load Average 过高问题

Linux 系统的 Load Average（负载均值）是衡量系统整体健康状态的重要指标。它反映了处于可运行状态或不可中断状态的进程数量。当 Load Average 远高于 CPU 核心数时，说明系统已经过载。

## 现象特征
1. **Load Average 飙升**：1分钟/5分钟/15分钟的平均负载值远超 CPU 核心数。
2. **CPU 使用率不一定高**：Load 高可能是因为大量进程处于 I/O 等待（不可中断状态），而非 CPU 密集计算。
3. **响应变慢**：系统整体变卡，SSH 登录困难，命令执行延迟增大。

## 常见原因
1. **CPU 密集型任务**：大量计算任务导致进程排队等待 CPU。
2. **I/O 阻塞**：磁盘 I/O 饱和或网络 I/O 等待（如 NFS 挂载点卡死）。
3. **内存不足导致交换**：如果内存耗尽，系统会频繁换页，导致大量进程处于不可中断状态。
4. **进程积压**：某个后台任务或定时 Job 启动过多子进程，导致进程数爆炸。

## 排查建议
- 使用 `top` 查看 CPU 和内存占用，定位高负载进程。
- 使用 `iostat` 检查磁盘 I/O 是否饱和。
- 检查是否有异常的后台任务或定时脚本在运行。
