# 干扰分析：HTTP 5xx 错误激增

HTTP 5xx 系列错误表示服务器端错误，是运维中最需要紧急处理的告警之一。5xx 错误的激增通常意味着服务出现了严重的异常，需要立即介入。

## 现象特征
1. **错误率突增**：监控面板显示 5xx 错误率从 0% 快速上升到 5% 甚至更高。
2. **影响范围广**：多个接口同时出现 5xx 错误，而非单一接口。
3. **伴随其他告警**：通常与 CPU、内存、磁盘、网络等资源告警同时出现。

## 常见原因
1. **服务崩溃**：应用进程 OOM 被杀、Panic 或崩溃。
2. **依赖故障**：下游服务（数据库、缓存、消息队列）不可用或超时。
3. **资源耗尽**：连接池耗尽、线程池耗尽、文件描述符耗尽。
4. **代码异常**：未捕获的全局异常、空指针异常、数组越界等。
5. **限流/熔断触发**：上游服务开启了限流或熔断，主动拒绝请求。

## 排查建议
1. 确认错误类型：502/503/504 的根因完全不同，需要分别处理。
2. 检查应用日志：重点搜索 ERROR 和 FATAL 级别日志。
3. 检查依赖服务：数据库、Redis、MQ 等下游组件是否正常。
4. 检查资源使用：CPU、内存、磁盘、网络、连接池是否达到上限。
5. 检查最近变更：是否有发版、配置变更或流量变更。
